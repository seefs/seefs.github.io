Set up a rush

5. Double stop loss: used for many small orders, centralized parameter modification.
Scripts\_OrderSetting\UpdatePairs_20_0.mq5

Application scenario: Large unilateral market is available. If it is a large unilateral side, it will be profitable, and if it is pulled up and down, it will be lost.
Step 1: Open an order first, long and short in both directions. Note that opening 1 large order is faster than opening 10 small orders.
Step 2: Don’t be stopped. Cancel the setup before 8:29 (Clear script). 8:30 20 seconds before the start of the setup (Init script). If the fluctuation is too fast before 8:30, you need to click this script repeatedly (Update stop loss).
Step 3: At 8:30, wait for the profit and stability.
Step 4: Click this script again before the swing pullback, and the profit is only a lot more, because the script restricts the stop loss to the pullback.
Step 5: Without stop loss, select a suitable position to close the position manually.
Function description: Stop loss -2, take profit unchanged.
Function 1: Hold the stop loss $2 from the current price. You can keep ordering.
Function 2: Stop loss cannot be pulled back when making profits.

Unilateral trend setup

10. Unilateral trend setting: do not write scripts, use the previous combination of functions.
Application scenario: How to configure parameters when the trend is halfway through.
Scenario 1: The market pauses and goes to eat. Set Take Profit +4 and Stop Loss -2. (10-2 stops with Init + Move)
Scenario 2: The market is weak to rebound. Set Take Profit +2 and Stop Loss -1. (Achieved with Init + Move stop loss 2–5 times)

Pre-Close Settings

11. Pre-closing settings: fixed parameters according to the actual application scenario.
The script is only available for XAUUSD, and the parameters for other symbols should be changed.

anti-reversal: A few scenes when you are ready to play, but you want to stay and watch.
Scripts\_OrderSetting\PreClosePairs_5_10.mq5

Application scenario: advantageous order, to the bottom, anti-breakthrough reversal.
Scenario 1: When the instantaneous reversal is 0.5, it will automatically appear.
Scenario 2: When reversing slowly, as long as the callback is limited, you can click a few more times to update the settings and prevent the order from being stopped.
Scenario 3: When slowly following the trend, tap a few more times. Keep the take profit at 1.0. A wave of market conditions brought it to the point of reversal.
Scenario 4: When the profit is estimated and the exit conditions are met, use this to expand the profit.

12. Trial and error:
Scripts\_OrderSetting\PreClosePairs_10_10.mq5

Application scenario: unoptimistic lists. You shouldn’t be in or it’s time to play.
Scenario 1: Place the wrong order, and admit it after losing a little.
Scenario 2: The order that is lucky to return to the original may be reversed quickly to prevent the loss from expanding.
Scenario 3: Just made a profit, not greedy.
Scenario 4: Take a risk to follow, and there must be a bottom limit for the drawdown.

13. Expand your profits:
Scripts\_OrderSetting\PreClosePairs_5_5.mq5

Application scenario: In some markets, a small fluctuation of 0.5 will be retraced, and no stop profit can be caught.
Scenario 1: The volatility is small in the morning, just take a profit of 0.5. Save yourself time.
Scenario 2: When there is a high probability of reversal, try to lose as little as possible. Take profit is swept away, which is usually a reversal signal.
Scenario 3: Expectations are small.

Magic number settings

partial liquidation: the function is too complicated, and it is troublesome to use, so I don’t consider it.
